breaking down the problem

  1. In order to solve a problem we often need to break it down from an initially complex task to a set of slightly less complex tasks that - together - solve the problem.
  2. There is RARELY only 1 way to solve a problem.
  3. When you break the initial task down 1 level, you will often have to break these 2nd level tasks down even further. This process might continue through several stages until the problem has been solved on a level that we feel we can make a start on
  4. breaking down a single subtask can be done rather independently from other tasks on that level

**the initial problem will often be too complex to solve all in one go. It's therefore necessary to break it down into smaller parts that are easier and more manageable

choice and repetition Ways to solve a problem:

  1. sequence of steps: "do this, then do this, then do this." the simplest way to solve a problem. However solving a problem isn't always that simple.

  2. making choices: We only do a dependent set of actions if we find that a certain condition is true. When making choices this is often a task we don't always want to perform. Another common scenario is to choose between 2 possible alternative actions depending on a condition. in this case only 1 of these will always be acted upon.

  3. doing things repeatedly: keep on doing something until it is just right. This is called indefintie/unbounded repition since we don't know how many times we need to repeat the action. Bounded repeition repeats the task for a fixed number of times to get it just right.
